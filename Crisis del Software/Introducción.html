Crisis del Software

Introducción


La crisis del software fue el evento principal que definió lo que hoy conocemos como ingeniería del software. Los problemas que se presentaban anteriormente con la construcción de los mismos (software), permitió sentar las bases para darle más calidad a cada fase del desarrollo y no cometer los mismos errores que se presentaban anteriormente. Para entender dicha crisis primero debemos comprender que es el software.
El software es un sistema operativo o programa que se implementa en un equipo para ejecutar tareas y operaciones por medio de instrucciones que codifica el desarrollador el base a lo que el cliente busca. Por decirlo de la mejor manera, un equipo sin software es obsoleto ya que este funciona como un “cerebro" que indica que funciones realizar y los pasos a seguir para alcanzar el objetivo final.
